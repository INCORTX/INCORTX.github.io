📷 Exercise 1: Sensor Resolution & Bit Depth (Theory)
คำถาม: เซนเซอร์ของกล้องถ่ายรูปตัวหนึ่งมีความละเอียด 12 ล้านพิกเซล (12 Megapixels = $4000 \times 3000$ พิกเซล) จงคำนวณหาขนาดของไฟล์ข้อมูลภาพดิบ (RAW Image Data) ในหน่วยเมกะไบต์ (MB) เมื่อบันทึกที่ความละเอียดระดับความลึกบิต (Bit Depth) 2 ค่า ได้แก่: 1) 12-bit RAW และ 2) 8-bit RGB (3 แชนแนล) และเปรียบเทียบวิเคราะห์ข้อแตกต่างเชิงพื้นที่เก็บข้อมูล

แนวคิด: ขนาดไฟล์ภาพดิบจะขึ้นอยู่กับจำนวนพิกเซล ความลึกของบิต (Bit Depth) และจำนวนแชนแนลสี โดยแปลงบิตข้อมูลเป็นไบต์โดยใช้ความสัมพันธ์ $1\text{ Byte} = 8\text{ bits}$

$\text{Size (bits)} = \text{Width} \times \text{Height} \times \text{Bit-depth} \times \text{Channels}$
  • 1. สำหรับการบันทึกแบบ 12-bit RAW (1 แชนแนลก่อนทำ Demosaicing):
    - จำนวนบิตรวม: $4,000 \times 3,000 \times 12\text{-bit} \times 1 = 144,000,000\text{ bits}$
    - แปลงเป็นไบต์: $\frac{144,000,000}{8} = 18,000,000\text{ Bytes}$
    - แปลงเป็นเมกะไบต์ (ฐาน 10): $18,000,000 / 10^6 =$ $18.0\text{ MB}$ (หรือ $17.17\text{ MiB}$ ในฐาน 2)
    ผลลัพธ์ 12-bit RAW: ขนาดไฟล์คือ $18.0\text{ MB}$ (เก็บข้อมูลระดับความสว่างได้ $2^{12} = 4,096$ ระดับต่อพิกเซล)
  • 2. สำหรับการบันทึกแบบ 8-bit RGB (3 แชนแนลสี):
    - จำนวนบิตรวม: $4,000 \times 3,000 \times 8\text{-bit} \times 3 = 288,000,000\text{ bits}$
    - แปลงเป็นไบต์: $\frac{288,000,000}{8} = 36,000,000\text{ Bytes}$
    - แปลงเป็นเมกะไบต์ (ฐาน 10): $36,000,000 / 10^6 =$ $36.0\text{ MB}$ (หรือ $34.33\text{ MiB}$ ในฐาน 2)
    ผลลัพธ์ 8-bit RGB: ขนาดไฟล์คือ $36.0\text{ MB}$ (เก็บข้อมูลสีผสมได้ $256^3 \approx 16.7$ ล้านสี)

บทวิเคราะห์ข้อเปรียบเทียบ:

แม้ว่าไฟล์ 12-bit RAW จะมีความลึกและความถูกต้องของข้อมูลแสงสูงกว่าแบบ 8-bit ถึง 16 เท่า (4,096 ระดับ เทียบกับ 256 ระดับ) แต่ขนาดไฟล์ RAW กลับมีขนาดเล็กกว่าไฟล์ RGB ถึง **ครึ่งหนึ่ง** ($18\text{ MB}$ vs $36\text{ MB}$)

สาเหตุหลักเกิดจาก **Bayer Filter Array** บนเซนเซอร์ ซึ่งมีฟิลเตอร์สีเพียงสีเดียวปกคลุมในแต่ละพิกเซล (ทำให้ RAW Image มีข้อมูลแชนแนลเดี่ยว ขนาด 12 บิตต่อพิกเซล) ส่วนไฟล์ 8-bit RGB เป็นภาพสีที่ผ่านกระบวนการอินเทอร์โพเลชันหาค่าสีรอบข้าง (Demosaicing) เรียบร้อยแล้ว ทำให้ต้องใช้พื้นที่จัดเก็บถึง 3 แชนเนลในทุกจุดพิกเซล ($8 \times 3 = 24$ บิตต่อพิกเซล) นั่นเอง

🎨 Exercise 2: Grayscale & Luminance Conversion (Theory)
คำถาม: ตัวอย่างการย้อมสีสไลด์เซลล์ทางพยาธิวิทยา (Pathology Stain) มีจุดสีที่วัดได้เป็น RGB = $(120, 200, 50)$ จงคำนวณหาค่า Grayscale (ความเข้มสว่าง $Y$) ด้วยวิธี: 1) Simple Average และ 2) BT.601 Luminance Formula และอธิบายว่าเหตุใดสูตรหลังจึงสอดคล้องกับการรับรู้ของมนุษย์ได้ดีกว่า

แนวคิดการคำนวณ:

กำหนดจุดพิกเซลสีเขียวตองอ่อน $R = 120, G = 200, B = 50$

  • 1. วิธี Simple Average (หาค่าเฉลี่ยเชิงเส้นธรรมดา):
    $Y_{\text{avg}} = \frac{R + G + B}{3} = \frac{120 + 200 + 50}{3} = \frac{370}{3} \approx 123.33$
  • 2. วิธี BT.601 Luminance Formula (การถ่วงน้ำหนักตามการตอบสนองของสายตา):
    $Y_{\text{BT601}} = 0.299 \cdot R + 0.587 \cdot G + 0.114 \cdot B$

    แทนค่าลงในสมการ:

    $Y_{\text{BT601}} = (0.299 \times 120) + (0.587 \times 200) + (0.114 \times 50) = 35.88 + 117.40 + 5.70 = 158.98$
    ผลลัพธ์: $Y_{\text{avg}} \approx 123.33$ | $Y_{\text{BT601}} \approx 159.0$ (ปัดค่าจำนวนเต็มได้ 159)

คำอธิบายเชิงวิทยาศาสตร์ของการรับรู้แสง (Human Visual Perception):

วิธี BT.601 ให้ค่า $Y \approx 159$ ซึ่งสูงกว่าวิธีหาค่าเฉลี่ย ($123.33$) อย่างเห็นได้ชัด เพราะสูตรนี้ถ่วงน้ำหนักสีเขียวไว้สูงสุดถึง **$58.7\%$** (ในขณะที่สีแดงได้รับ $29.9\%$ และสีน้ำเงินเพียง $11.4\%$)

ในเชิงชีววิทยา จอประสาทตาของมนุษย์ (Retina) มีเซลล์รับแสงรูปกรวย (Cone Cells) 3 ชนิด โดยเซลล์รับแสงสีเขียว (M-cones) และเซลล์รับแสงสีแดง (L-cones) มีจำนวนหนาแน่นและตอบสนองได้ดีกว่าเซลล์รับแสงสีน้ำเงิน (S-cones) มาก ทำให้เมื่อเรามองสีเขียวที่มีกำลังพลังงานคลื่นเท่ากับสีน้ำเงิน สายตามนุษย์จะตีความว่าสีเขียว **สว่างกว่ามาก**

ดังนั้น การแปลงภาพสีเป็นเกรย์สเกลในคอมพิวเตอร์วิชันจึงนิยมใช้สูตรถ่วงน้ำหนักเพื่อรักษาความต่างระดับความสว่างในมุมมองที่เลียนแบบมนุษย์มากที่สุด หากใช้ค่าเฉลี่ยธรรมดา ภาพที่ได้จะมีโทนสว่างที่หม่นและขัดแย้งกับความเป็นจริงของสายตา

📊 Exercise 3: Image Histograms Analysis (Theory)
คำถาม: จงวิเคราะห์ลักษณะและรูปทรงของกราฟฮิสโทแกรม (Histogram) ต่อไปนี้ และระบุว่ารูปทรงดังกล่าวตรงกับลักษณะทางกายภาพของภาพแบบใด (ระหว่าง: ภาพสว่างเกินไป (Overexposed), ภาพมืดเกินไป (Underexposed), ภาพคอนทราสต์ต่ำ (Low Contrast), และภาพคอนทราสต์สูง (High Contrast)):
  • ก) กราฟมีจุดยอดแหลมสูงมากกระจุกตัวอยู่เฉพาะในย่านค่าสีต่ำ (ใกล้ 0) และย่านอื่น ๆ แทบไม่มีค่าเลย
  • ข) กราฟกระจายตัวแบนราบเต็มพื้นที่ตั้งแต่ 0 ถึง 255 และมีจุดยอดสูงขวาสุดและซ้ายสุดแบบสองขั้ว (Bimodal)
  • ค) กราฟกระจุกตัวเป็นแถบแคบ ๆ อยู่ตรงกึ่งกลาง (ใกล้ 128) โดยไม่มีค่าสีที่เข้าใกล้ 0 หรือ 255 เลย
  • ง) กราฟมีจุดยอดกระจุกตัวหนาแน่นอยู่เฉพาะในย่านค่าสีสูง (ใกล้ 255)

ผลการวิเคราะห์และจับคู่ทางกายภาพ:

  • ก) ตรงกับ: ภาพมืดเกินไป (Underexposed)

    เหตุผล: แกนนอนของฮิสโทแกรมแสดงระดับความสว่างจาก 0 (ดำสนิท) ไปจนถึง 255 (ขาวจัด) การที่ข้อมูลกระจุกตัวอยู่ที่ฝั่งซ้ายของกราฟ แปลว่าเกือบทุกพิกเซลในภาพมีค่าความสว่างต่ำมากหรือมืด

  • ข) ตรงกับ: ภาพคอนทราสต์สูง (High Contrast)

    เหตุผล: ภาพที่มีคอนทราสต์สูงจะมีเฉดสีดำเข้มและสีขาวจ้าชัดเจน และมีการกระจายตัวของระดับสีที่หลากหลาย ทำให้มีส่วนประกอบที่มืดสนิทและสว่างสุดอยู่สูง เกิดเป็นยอดเขาโด่งที่บริเวณฝั่งซ้ายสุดและขวาสุดของกราฟ

  • ค) ตรงกับ: ภาพคอนทราสต์ต่ำ (Low Contrast)

    เหตุผล: คอนทราสต์ต่ำหมายความว่าแต่ละพิกเซลมีค่าสีเทาใกล้เคียงกัน ไม่มีส่วนที่สว่างจัดหรือมืดจัด ส่งผลให้กราฟฮิสโทแกรมมีลักษณะหดตัวแคบเป็นรูประฆังคว่ำกระจุกตัวอยู่เฉพาะบริเวณกึ่งกลางภาพ

  • ง) ตรงกับ: ภาพสว่างเกินไป (Overexposed)

    เหตุผล: พิกเซลจำนวนมากมีค่าสีสูงเทกองอยู่ใกล้ขอบขวาสุด (ใกล้ 255) แสดงว่าแสงล้นทะลุขีดจำกัดเซนเซอร์ รายละเอียดส่วนสว่างสูญหายกลายเป็นสีขาวจ้า

💻 Exercise 4: NumPy Channel Swapping & Cropping (Code)
คำถาม: จงเขียนฟังก์ชันภาษา Python โดยใช้เพียงไลบรารี `numpy` เพื่อรับภาพสีอินพุต `img` ขนาด $(H, W, 3)$ แล้วทำการ: 1) สลับช่องสัญญาณสีจาก RGB เป็น BGR โดยใช้การสไลซ์อาเรย์ (Array Slicing) และ 2) ครอป (Crop) ตัดเฉพาะพื้นที่รูปสี่เหลี่ยมจัตุรัสขนาด $100 \times 100$ พิกเซล จากบริเวณกึ่งกลางของภาพนั้น

ตัวอย่างฟังก์ชันและคำอธิบาย:

import numpy as np

def process_cv_week1(img):
    # ตรวจสอบขนาดว่าภาพมีขนาดใหญ่กว่า 100x100 หรือไม่
    H, W, C = img.shape
    if H < 100 or W < 100:
        raise ValueError("Image size must be at least 100x100 pixels.")

    # 1. สลับช่องสีจาก RGB -> BGR ด้วยการสไลซ์แกนสุดท้าย (แกนที่ 2) ย้อนกลับ
    bgr_img = img[:, :, ::-1]

    # 2. คำนวณหาจุดเริ่มต้นในการครอปตัดกึ่งกลางภาพ
    start_y = (H - 100) // 2
    start_x = (W - 100) // 2

    # ครอปภาพตัดกึ่งกลางของ BGR Image
    center_crop = bgr_img[start_y:start_y+100, start_x:start_x+100, :]
    return center_crop

วิเคราะห์กลไกโค้ด (Code Mechanics):

- **`img[:, :, ::-1]`** คือการดึงค่าพิกเซลทั้งหมดในแกน $H$ และ $W$ แต่กลับทิศทางลำดับในแกนสี ส่งผลให้ดัชนีช่องสีเดิม $[0, 1, 2]$ (R, G, B) ถูกดึงออกมาเรียงใหม่เป็น $[2, 1, 0]$ (B, G, R) ทำได้ในระดับ Memory View โดยไม่ต้องคัดลอกข้อมูลใหม่ (O(1) complexity)

- **`start_y` และ `start_x`** ใช้หลักการลบความกว้างความสูงของเฟรมเป้าหมายออก แล้วหารสองปัดเศษลงเพื่อให้พิกัดเริ่มต้นมีความสมดุลย์กึ่งกลางภาพที่สุด จากนั้นใช้การสไลซ์แกนเพื่อดึงเฉพาะพิกเซลช่วงเป้าหมายออกมา

🛠️ Exercise 5: Brightness & Contrast Adjustment (Code)
คำถาม: จงเขียนฟังก์ชันภาษา Python `adjust_image(img, alpha, beta)` โดยใช้เพียงไลบรารี `numpy` เพื่อปรับแต่งความสว่างและคอนทราสต์ของภาพสี `img` (ประเภทข้อมูล `uint8`) ตามสูตร $I_{\text{new}} = \alpha \cdot I + \beta$ โดยฟังก์ชันต้องป้องกันไม่ให้เกิดความคลาดเคลื่อนล้นช่วง (Overflow/Underflow) และส่งคืนภาพกลับมาเป็นประเภท `uint8` อย่างถูกต้อง

ฟังก์ชันการปรับความสว่างและคอนทราสต์ที่ถูกต้อง:

import numpy as np

def adjust_image(img, alpha, beta):
    # 1. แปลงประเภทรูปภาพดั้งเดิมเป็น float เพื่อป้องกันค่าล้นระหว่างการคูณและบวก
    img_float = img.astype(np.float32)

    # 2. ปรับใช้สูตรเชิงเส้น: alpha คือตัวคูณคอนทราสต์, beta คือค่าเพิ่มความสว่าง
    adjusted = alpha * img_float + beta

    # 3. จำกัดขอบเขตค่าพิกเซลด้วย np.clip ให้อยู่ในช่วงสี 0-255 เสมอ (ป้องกัน Overflow)
    clipped = np.clip(adjusted, 0, 255)

    # 4. แปลงประเภทข้อมูลกลับเป็นจำนวนเต็ม 8 บิตไม่มีเครื่องหมาย (uint8) ก่อนส่งคืน
    return clipped.astype(np.uint8)

ประเด็นระวังเชิงโปรแกรมมิ่ง (Crucial Programming Trap):

หากคำนวณบนข้อมูลประเภท uint8 ตรง ๆ (เช่น `img * 1.5 + 20`) ค่าที่เกิน 255 จะเกิดอาการ **Overflow** ทันทีเนื่องจากความจุหน่วยความจำ 8-bit เก็บได้ไม่เกิน 255 (เช่น ค่า 256 จะปัดรอบกลับมาเป็น 0 ซึ่งทำให้ส่วนที่สว่างจ้ากลายเป็นจุดสีดำมืดผิดรูปทันที)

การหลีกเลี่ยงข้อผิดพลาดนี้จำเป็นต้องแปลงไทป์อัปสเกลข้อมูลให้เป็นทศนิยมลอยตัว float32 เพื่อรองรับค่าชั่วคราวที่อาจเกิน 255 หรือติดลบ จากนั้นเรียกใช้ฟังก์ชันควบคุมขอบเขต np.clip แล้วค่อยดาวน์แคสต์ประเภทข้อมูลกลับมาเป็น uint8 อีกครั้ง

🔬 Exercise 6: Stain Segmentation Pipeline (Application)
คำถาม: จงเขียนบล็อกโค้ด NumPy สกัดแชนเนลสีเขียว (Green channel), ทำการปรับระดับคอนทราสต์ให้ชัดขึ้น, ประยุกต์ใช้ค่าเกณฑ์ (Thresholding) ที่ระดับความเข้ม 120 เพื่อสร้างหน้ากากรูปภาพไบนารี (Binary Mask) และคำนวณหาเปอร์เซ็นต์พื้นที่ของเซลล์ที่ถูกย้อมเข้มเมื่อเทียบกับพื้นที่รูปภาพทั้งหมด

ตัวอย่างทาสก์โค้ดสำหรับสกัดพื้นที่เซลล์:

import numpy as np

def segment_green_cells(img_rgb):
    # 1. สกัดดึงแชนเนลสีเขียว (Green channel อยู่ดัชนีที่ 1 ในมิติ RGB)
    green_ch = img_rgb[:, :, 1]

    # 2. ปรับคอนทราสต์แชนเนลนี้ขึ้น 1.5 เท่า ด้วยการแปลงเป็น float และตัดขอบขอบเขต
    enhanced = np.clip(1.5 * green_ch.astype(np.float32), 0, 255).astype(np.uint8)

    # 3. กำหนดค่าเกณฑ์ (Thresholding) ที่ความสว่าง 120
    # ค่าพิกเซลที่เข้มกว่า 120 จะกลายเป็น 1 (สีขาว) และส่วนอื่นเป็น 0 (สีดำ)
    binary_mask = (enhanced > 120).astype(np.uint8)

    # 4. คำนวณสัดส่วนพื้นที่เซลล์สีเขียวเป็นเปอร์เซ็นต์
    cell_area = np.sum(binary_mask) # นับเฉพาะพิกเซลที่เป็น 1
    total_area = binary_mask.size # จำนวนพิกเซลรวมในภาพ (H * W)
    cell_percentage = (cell_area / total_area) * 100.0

    return binary_mask, cell_percentage

คำวิจารณ์เชิงประยุกต์ใช้งาน (Practical Insight):

การสกัดเลือกเฉพาะแชนเนลสีเดี่ยวสีใดสีหนึ่ง (ในกรณีนี้คือสีเขียว) เป็นหนึ่งในเทคนิคการเพิ่มความแตกต่าง (Contrast Enhancement) ที่เรียบง่ายแต่ทรงพลังในทางการแพทย์ เนื่องจากรูปภาพสีเขียวจะมีจุดเด่นที่สุดในช่องสัญญาณสีนั้น ทำให้การทำ Thresholding สามารถทำได้อย่างแม่นยำและประหยัดการคำนวณมากกว่าการนำเวกเตอร์ 3 ช่องมาระบายขอบในคราวเดียวกัน

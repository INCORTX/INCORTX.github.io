The Data Science Process

A systematic four-step approach to solving problems with data.

1. Define

Articulate the problem clearly. What question are you answering?

2. Collect

Gather relevant data from DBs, APIs, surveys, or sensors.

3. Analyze

Clean data, explore patterns, and build predictive models.

4. Communicate

Present findings via dashboards or reports to stakeholders.

Real World Example: Predicting customer churn to improve retention strategies.

Data Preparation: The Reality

The "80/20" Rule of AI

Most people think AI is about coding models. In reality, it's about data.

The Cooking Analogy

Model = Cooking Pot | Data = Ingredients
Even a Michelin-star chef cannot make good food from rotten ingredients.

Real Impact (Project X)

  • Data cleaning 2 weeks → Accuracy +15%
  • Model tuning 1 week → Accuracy +2%

Context: Signal vs. Noise

The Core Objective

Every dataset consists of the pattern we want (Signal) and errors (Noise).

y = f ( x ) + ϵ
  • f(x): The Signal (e.g., actual face pattern)
  • ε: The Noise (blur, bad lighting)

Goal: Minimize ε so the model learns the true f(x).

Concept Distinction:
• Here: Minimize Unwanted Noise (Errors).
• Augmentation (Later): Add Controlled Variance (Robustness).

The Strategic Framework

Transforming raw data into robust model inputs requires a structured, multi-stage approach.

Phase 1: Prep

Essential Preprocessing

Normalization & Standardization

Phase 2: Split

Separation Strategy

Random, Stratified, or Group

Phase 3: Refine

Training Set Refinement

Augmentation & Resampling

Phase 1: Essential Preprocessing

Making Data "Standard"

1. Numerical Data (e.g., Salary)

Scale to 0-1 range (Normalization) or transform to Mean=0, Std=1 (Standardization).
Why? Models are biased by large numbers.

2. Text Data (NLP/LLM)

Cleaning, Lowercasing, Tokenization.
Why? "Hello!" vs "hello" should match.

Stack: torchvision | NLTK / spaCy

Phase 2: The Separation Strategy

Choosing the Right Logic

Instead of a linear path, separating data requires a strategic decision based on your dataset's characteristics.

Navigator:
Follow the decision tree to find the correct strategy for your project.

Strategy A: Random Split

The Standard Approach

Randomly shuffling data into Train (70%), Valid (15%), Test (15%).

Best for: Large, Balanced Data.

Problem: Imbalanced Data

Example: 1000 samples, 10 defects
→ Test set might get 0 defects!

Solution: Stratified Sampling

Strategy B: Stratified Split

Solving the "Rare Case" Problem

Problem: Random shuffle might leave ZERO defects in the Test set for imbalanced data.

Solution: Force the split to maintain the same ratio.

1% Defects in Train ↔ 1% Defects in Test

Stack: sklearn.model_selection.StratifiedKFold

Strategy C: Group Split

Preventing Data Leakage

Problem: Mixing photos of the same person in Train/Test creates memorization.

WRONG: Person A in Train & Test
Result: Model memorizes A. Fails on new people.
CORRECT: Person A in Train Only
Result: Model learns general patterns. WORKS!
Real-World Disaster Company X: Test Acc 95% → Prod 60%
Reason: Data Leakage!
Cost: $500K wasted.

Phase 3: Data Augmentation

Multiplying Data with Logic

1. Computer Vision (Images):
Rotation, Flip, Zoom, Noise.

2. NLP (Text):
Synonym Replacement, Back-Translation.

CRITICAL RULES

✅ Augment Training Set ONLY
❌ Never Touch Test Set!

Stack: Albumentations | nlpaug

Phase 3 (Cont.): Data Resampling

Balancing the Scales

When you have extreme class imbalance, augmentation might not be enough. You need to reshape the dataset.

Downsampling (Under-sampling)

Removing samples from the Majority class.

Oversampling

Duplicating samples from the Minority class (SMOTE).

CAUTION: DOWNSAMPLING

Downsampling throws away data (signal). Use only when you have massive datasets (millions of rows) with high redundancy.

Common Mistakes to Avoid

Augmenting the Test Set

Never apply rotation/noise to test data. It must represent reality, not synthetic variations.

Ignoring Class Distribution

Failing to check if your test set actually contains the rare classes (defects) you need.

Separate Normalization

Wrong: Calc Mean/Std on Test.
Right: Use Mean/Std from Train to transform Test.

Feature Leakage

Creating features (like "Group Average") using the whole dataset before splitting.

Final Recap & Conclusion

A structured approach to data preparation significantly improves model reliability.

1. Normalize (Phase 1)

Scale inputs and clean text. Always mandatory.

2. Strategize Split (Phase 2)

Random, Stratified, or Group split based on data characteristics.

3. Refine Train Set (Phase 3)

Apply Augmentation and Resampling to the Training Set only.

Golden Rule

"Garbage In, Garbage Out"
